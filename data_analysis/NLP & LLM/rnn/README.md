README: RNN для классификации отзывов магазинов

Catboost (0.62) и RNN (0.60) похуже трансформеров справлялись с этой задачей 

📊 Задача
Классификация текстовых отзывов о магазинах "Пятёрочка" на 5 классов (оценки от 1 до 5 звезд).
Целевая метрика: F1-score (weighted).
Цель: Превысить hard baseline (F1 > 0.68).

🎯 Наш подход: Улучшенная двунаправленная LSTM с механизмом внимания
Ключевые особенности решения:
BiLSTM (Bidirectional LSTM) — читает текст в двух направлениях, улавливая контекст как слева направо, так и справа налево

Механизм внимания (Attention) — модель учится фокусироваться на важных словах в отзыве (например, "ужасно", "отлично", "грязно")

Предобученные эмбеддинги — используем word2vec-ruscorpora-300 (300-мерные вектора для русского языка)

Label Smoothing — сглаживание меток для борьбы с переобучением

Ранняя остановка — автоматическое прекращение обучения при ухудшении качества

Cosine Annealing — динамическое изменение скорости обучения для лучшей сходимости

Почему RNN, а не Transformer?
Данных относительно немного (~48k отзывов)

RNN требует меньше вычислительных ресурсов

При правильной настройке RNN показывает результаты, сравнимые с BERT на небольших датасетах

📄 Описание файлов
dataset.py — подготовка данных
Классы и функции:

Компонент	Назначение
AdvancedTokenizer	Разбивает текст на слова, удаляет пунктуацию, стоп-слова, приводит к нижнему регистру
Vocab	Строит словарь частотных слов (до 50000 слов), заменяет редкие на <UNK>
TextDataset	PyTorch Dataset для загрузки данных, поддерживает паддинг и обрезку длинных текстов
train_test_split	Стратифицированное разделение на train/val с сохранением пропорций классов
Ключевая логика:

python
# Токенизация текста
"Очень понравилось! Были в начале марта" → ["очень", "понравилось", "были", "начале", "марта"]

# Векторизация (слова → индексы)
["очень", "понравилось"] → [1542, 8913, 2]  # 2 = <EOS>
model.py — архитектура нейросети
Компоненты модели:

text
Входной текст (индексы)
       ↓
Embedding Layer (300-мерные вектора)
       ↓
Dropout (0.2)
       ↓
BiLSTM (3 слоя, 256 нейронов, dropout 0.3)
       ↓
Attention Mechanism (взвешенное суммирование)
       ↓
Dropout (0.4)
       ↓
MLP Classifier (128 → 64 → 5 классов)
       ↓
Выход (логиты для 5 классов)
Ключевые классы:

Класс	Назначение
prepare_emb_matrix()	Загружает предобученные эмбеддинги для слов из словаря
Attention	Механизм внимания — вычисляет важность каждого слова
RecurrentClassifier	Основная модель, объединяющая все компоненты
Почему Attention важен?
В отзывах ключевые слова могут быть в любом месте. Attention позволяет модели "видеть" всё предложение целиком и выделять самые информативные слова.

trainer.py — обучение и валидация
Классы и функции:

Компонент	Назначение
EarlyStopping	Останавливает обучение, если F1 не растёт 7 эпох, восстанавливает лучшие веса
Trainer	Управляет процессом обучения: forward, backward, оптимизация, логирование
Ключевые техники обучения:

AdamW оптимизатор — улучшенная версия Adam с корректной L2-регуляризацией

Label Smoothing — вместо [0,0,1,0,0] используем [0.05, 0.05, 0.8, 0.05, 0.05] (снижает переобучение)

Cosine Annealing — скорость обучения циклически уменьшается и возрастает

Gradient Clipping — ограничивает градиенты для стабильности

Weighted Random Sampler — балансирует классы при обучении (класс 5 встречается чаще)

rnn.ipynb — основной файл запуска
Пошаговый пайплайн:

Шаг	Что делает
1	Загружает данные, выводит статистику
2	Очищает текст (удаляет эмодзи, нормализует числа)
3	Кодирует метки (5→4, т.к. модель ожидает 0-4)
4	Токенизирует и строит словарь
5	Разделяет на train/val (85%/15%)
6	Создаёт DataLoader'ы с балансировкой классов
7	Загружает эмбеддинги word2vec (300МБ, ~1-2 минуты)
8	Инициализирует модель и тренера
9	Обучает модель (30 эпох, ранняя остановка)
10	Оценивает на валидации (F1, confusion matrix)
11	Предсказывает на тесте
12	Создаёт submission файл
13	Сохраняет модель и вероятности для ансамбля
🔧 Гиперпараметры (оптимальные)
python
# Архитектура
"cell_type": "LSTM"           # Тип рекуррентной ячейки
"hidden_size": 256            # Размер скрытого состояния
"num_layers": 3               # Количество слоёв
"bidirectional": True         # Двунаправленная
"use_attention": True         # Механизм внимания
"cell_dropout": 0.3           # Dropout между слоями LSTM
"out_dropout": 0.4            # Dropout в классификаторе

# Обучение
"lr": 0.001                   # Learning rate
"weight_decay": 1e-4          # L2 регуляризация
"batch_size": 64              # Размер батча
"n_epochs": 30                # Максимум эпох
"patience": 7                 # Ранняя остановка
Почему именно эти значения?

hidden_size=256 — золотая середина: меньше — теряем качество, больше — переобучение

num_layers=3 — достаточно для улавливания сложных зависимостей

dropout=0.4 — агрессивная регуляризация из-за небольшого датасета
